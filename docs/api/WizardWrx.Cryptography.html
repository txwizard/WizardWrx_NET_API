DigestFile

Static methods for computing message digests for files, using the most common algorithms.

iMPORTANT: Since files are read in binary, they are loaded directly into the hash algorithm as byte arrays. This means that the digest of a file of ASCII characters and the hash of the file contents read into a CLR string object, which is a string of Unicode characters, will differ. The reason for this is that the string of Unicode characters yields a different byte stream than the byte stream that came from reading the file in binary mode.
